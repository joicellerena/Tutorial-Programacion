🔄 Ciclos (Bucles) en Java

📖 Introducción

Cuando comenzamos a programar, muchas veces necesitamos realizar una misma tarea varias veces. Por ejemplo, mostrar los números del 1 al 100, calcular las notas de varios estudiantes o recorrer una lista de datos.

Si intentáramos hacer esto escribiendo una instrucción para cada caso, el programa sería muy largo y difícil de mantener. Para solucionar este problema existen los ciclos o bucles.

Un ciclo es una estructura de control que permite repetir un bloque de instrucciones varias veces mientras se cumpla una condición determinada.

🌎 Ejemplo de la vida real

Imagina que eres profesor y necesitas pasar asistencia a 30 estudiantes. En lugar de repetir la misma acción 30 veces manualmente, realizas el mismo proceso hasta terminar la lista. Los ciclos funcionan de manera similar.

🎯 ¿Para qué sirven los ciclos?

  • Automatizar tareas repetitivas.
  • Ahorrar tiempo al programador.
  • Reducir la cantidad de código.
  • Procesar grandes cantidades de datos.
  • Recorrer vectores, matrices y listas.

📚 Tipos de ciclos en Java

  • for
  • while
  • do-while

🔹 Ciclo For

El ciclo for se utiliza cuando conocemos exactamente cuántas veces queremos repetir una acción. 


for(int i = 1; i <= 5; i++){
    System.out.println(i);
}

📝 Explicación línea por línea

  • int i = 1: crea una variable llamada i y le asigna el valor inicial 1.
  • i <= 5: indica que el ciclo continuará mientras i sea menor o igual a 5.
  • i++: incrementa el valor de i en una unidad cada vez que se ejecuta el ciclo.
  • System.out.println(i): muestra el valor actual de i en la pantalla.

📤 Resultado

1
2
3
4
5

⚠️ Error común

Muchos principiantes olvidan incrementar la variable del ciclo. Esto provoca que el ciclo nunca termine y el programa se ejecute indefinidamente.

🎯 Ejercicio Propuesto

Crear un programa que muestre los números del 1 al 20 utilizando un ciclo for.

✅ Solución


for(int i = 1; i <= 20; i++){
    System.out.println(i);
}

🏆 Conclusión

Los ciclos son una de las herramientas más importantes de la programación porque permiten automatizar tareas repetitivas y crear programas más eficientes.
